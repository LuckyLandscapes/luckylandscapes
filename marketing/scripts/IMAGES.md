# Adding & optimizing images

The site's images are split into two folders:

- `marketing/source-images/` — high-res originals (your camera output, drone shots, anything large). **This is the master copy.** Don't edit anything in `public/images/` directly.
- `marketing/public/images/` — web-ready, compressed versions. Generated by the optimizer. This is what the deployed site serves.

## Adding a new gallery photo

1. Drop the original into the right folder under `source-images/`. Examples:
   ```
   source-images/lawncare/9.jpg          # next lawn-care photo
   source-images/retainingwall/6.webp    # next retaining wall photo
   source-images/team/sean-2026.webp     # team headshot
   ```
   Use the filename pattern that already exists in that folder so the gallery picks it up.

2. Re-run the optimizer:
   ```bash
   cd marketing
   npm run optimize-images
   ```
   It reads from `source-images/`, writes optimized versions into `public/images/` at matching paths. You'll see per-file size deltas in the console.

3. If the new photo should appear in the homepage gallery or lightbox, edit the `projectData` array near the top of [`main.js`](../main.js) — add the path string under the right project's `images: []`.

4. Commit both the source-image AND the optimized output. Both are tracked in git.

## Adding a service hero or banner

Service hero photos go directly into `source-images/` (top level) or into a folder. Update the `<img src="...">` reference in the relevant HTML page (e.g. `services/lawn-care.html`).

## Per-folder quality profile

Quality is set per folder in [`optimize-images.js`](optimize-images.js) via the `PROFILES` object. Defaults work for almost everything; only edit if you need higher fidelity for a specific folder.

| Folder / file       | Max width | Quality | Notes                     |
|---------------------|-----------|---------|---------------------------|
| `Icon.png`          | 512px     | 90      | Logo, kept as PNG         |
| `og-card.png`       | 1200x630  | 82      | Re-encoded as JPEG inside |
| `banner.jpg`        | 1920px    | 72      | Hero banner               |
| `team/`             | 900px     | 80      | Team headshots            |
| Gallery folders     | 1600px    | 76      | All landscape photos      |
| `favicons/`, `favicon/` | (passthrough) |  | No re-encoding            |
| (any other folder)  | 1600px    | 75      | Default                   |

## Force re-encode

The optimizer skips files where the destination is already smaller and newer than the source. To force a full pass (e.g. after changing a quality setting):

```bash
node scripts/optimize-images.js --force
```

To process just one file:

```bash
node scripts/optimize-images.js source-images/lawncare/9.jpg
```

## Image format choice

- **`.webp`** — default for photos. Universal browser support, ~30% smaller than JPEG at equivalent quality.
- **`.jpg`** — only for files that need legacy compat. The optimizer accepts JPG input/output transparently.
- **`.png`** — only for the logo and graphics with transparency. Photos exported as PNG will get re-encoded back to PNG (palette-mode) but you'll get a much better result by converting them to WebP first.

## What happens if I forget to run the optimizer?

The build still works — it just ships the unoptimized version of whatever's in `public/images/`. You'll see it in CF Pages bandwidth/load times. CI doesn't auto-run the optimizer.
