# Lucky Landscapes — Business context for Claude

These docs give Claude the human / business context that can't be derived from the code. Fill in what you can — even rough notes are useful. Anything left as a `TODO:` or blank is fine; Claude will know it's unknown.

| File | What goes here |
|---|---|
| [`company.md`](company.md) | Identity: who we are, mission, brand voice, service area, team |
| [`services-pricing.md`](services-pricing.md) | What we sell, how we price it, common upsells |
| [`operations.md`](operations.md) | How the work actually gets done — crew, equipment, vendors, seasonal cadence |
| [`finances.md`](finances.md) | Revenue model, cost structure, current numbers (drop CSV/PDF exports here) |
| [`strategy.md`](strategy.md) | Current goals, constraints, what "the right move" looks like |

## How to keep these useful
- Update `finances.md` whenever you pull a new P&L or want a strategic conversation grounded in real numbers.
- Update `strategy.md` when goals shift (start of season, after a big win/loss, when priorities change).
- `company.md`, `services-pricing.md`, and `operations.md` change slowly — revisit a couple of times a year.
- For numbers that change weekly (jobs in pipeline, AR balance), the luckyapp database is the source of truth — don't duplicate it here.
